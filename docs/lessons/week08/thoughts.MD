### 🎲 上帝擲骰子：程式裡的混沌 (Chaos)

這週的主題是 **隨機 (Random)**。
老師問：「如果遊戲裡的寶箱都在同一個位置，好玩嗎？」
廢話，當然不好玩。這就是為什麼我們需要 `import random`。

我原本以為寫隨機很簡單，不就是 `random.randint(0, 800)` 嗎？
結果老師說這是 **「新手村最大的陷阱」**。
如果食物出現在 $x=43$，但我的蛇每次走 40 格 (0, 40, 80...)，我也許會跟食物擦肩而過，永遠吃不到它！
這就是傳說中的 **幽靈蘋果 (Ghost Apple)** BUG。

---

### 📐 網格對齊 (Grid Snapping)：秩序中的混亂

為了解決幽靈蘋果，我們學到了一個超重要的數學公式：

```python
格子編號 = random.randint(0, 19)
真實座標 = 格子編號 * 40
```

這個邏輯太神了！先算出有幾格，選一格，再乘回去。
這樣算出來的數字永遠是 40 的倍數 (0, 40, 80, 120...)，保證食物會乖乖待在格子正中間。
這讓我想到以前玩 RPG 製作大師，角色也是一格一格走的。原來這叫 **Grid-Based Logic**。

---

### 🌍 Global 的困惑

今天還有一個新語法讓我卡關很久：`global`。
在 `spawn_food()` 函數裡，如果直接寫 `food_x = ...`，Python 會以為我在創造一個新的區域變數，結果外面的 `food_x` 根本沒變。
必須加上 `global food_x`，才能真正改到外面的變數。
雖然覺得有點龜毛，但老師說這是為了安全，怕我們不小心改壞東西。

---

### 💎 掉寶率 (Drop Rate)：我也能設計轉蛋？

最讓我有 Vibe 的是進階題的「毒蘋果」。
老師教我們用隨機數來決定食物的顏色：
```python
if random.randint(1, 100) <= 10:
    color = PURPLE  # 10% 機率出現稀有怪！
```
這不就是手遊抽卡的 **SSR 機率** 嗎？！
我馬上把機率改成 1%，然後把顏色改成 **金色 (GOLD)**。
當那顆金色蘋果跳出來的時候，我整個人都燃起來了！原來設計遊戲數值的感覺這麼爽。
