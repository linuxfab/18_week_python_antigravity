# 第 02 週 | 顛倒世界：打破座標的常識

> **給教練的戰術板 (Coach's Corner)**：
> 這週有個大魔王叫 **「Y 軸」**。
> 國中數學告訴學生「Y 軸往上是正」，但電腦繪圖偏要跟你唱反調：「Y 軸往**下**才是正」。
> 這不是故意刁難，是因為以前的電視是從左上角開始掃描的。
> 請善用 `demo.py` 的滑鼠點擊功能，讓學生親眼「看到」這個殘酷的真相。座標不再是考卷上的數字，而是螢幕上的像素點。

## 📊 簡報精華 (Slide Highlights)

### Slide 1: 歡迎來到像素宇宙 (Pixel Universe)
*   **想像一下**：你的螢幕其實是由百萬個微型燈泡組成的矩陣。
*   **問題**：我要怎麼精準命令「第 3 排第 5 個燈泡」亮起來？
*   **答案**：座標 (Coordinate) —— `(x, y)`。這就是我們與螢幕溝通的地址。

### Slide 2: 顛倒的世界 (The Upside Down)
*   **數學課的騙局**：數學課本說 Y 軸往上是正數 (高度)。
*   **程式課的真相**：在電腦裡，Y 軸往**下**才是正數 (深度/行數)。
*   **記憶駭客**：就像讀書一樣，你的視線是從 **左上角** 開始，一行一行 **往下** 讀的。這就是電腦繪圖的順序。

### Slide 3: 顏色的煉金術 (RGB Alchemy)
*   **光的三原色**：Red (紅), Green (綠), Blue (藍)。
*   **能量值**：0 (全黑/關燈) ~ 255 (全亮/開燈)。
*   **Vibe Coding 挑戰**：
    *   (255, 0, 0) = 熱血紅。
    *   (0, 255, 0) = 駭客綠。
    *   (255, 255, 0) = ? (提示：紅光 + 綠光 = 黃光，這不是美術顏料喔！)

### Slide 4: 召喚第一條蛇 (Summoning the Snake Head)
*   **核心咒語**：`pygame.draw.rect(screen, color, rect)`
*   **Rect 四大天王**：`(x, y, width, height)`
*   **定位挑戰**：如何把一個方塊完美置中？
    *   不要用猜的！用算的！
    *   中心點 `center` = (螢幕寬/2, 螢幕高/2)
    *   左上角 `topleft` = (中心點x - 方塊寬/2, 中心點y - 方塊高/2)

## 📝 程式碼邏輯拆解 (Code Breakdown)

### 1. `Rect` 物件：隱形的框框
我們不再硬寫一堆數字，而是把它包裝成一個物件：
```python
snake_head = pygame.Rect(rect_x, rect_y, BLOCK_SIZE, BLOCK_SIZE)
```
這一行還沒有畫出任何東西喔！這只是在記憶體裡切出一塊「隱形的領地」。直到 `pygame.draw` 呼叫它，它才會顯形。

### 2. `pygame.draw.rect` 的兩種型態
*   `width=0` (預設)：**實心填滿**。適合畫蛇的身體。
*   `width=2`：**空心邊框**。適合畫選取框、Hitbox (判定框) 或者裝飾。
*   本週範例我們同時用了這兩招，讓蛇頭看起來更有層次，不再只是個無聊的色塊。

### 3. Debug 神器：滑鼠座標偵測
```python
if event.type == pygame.MOUSEBUTTONDOWN:
    print(f"Mouse Clicked at: {mouse_pos}")
```
這行程式碼價值連城！
當你搞不清楚座標在哪時，點一下螢幕，終端機馬上告訴你答案。這就是 **Interactive Debugging**。
